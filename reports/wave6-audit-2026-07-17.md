# Wave 6 — independent audit & gap diagnosis — 2026-07-17

Audit of the **live** production site (gaurav-gandhi.vercel.app, main @ `3bed3d6`, wave 5
merged) at 1440/768/390px, all sections and interactive states, **independent of prior wave
reports' self-assessments**. Benchmarked against five external references loaded and
screenshotted the same day — chosen for diversity, not resemblance:

| Reference | Archetype | What it proves |
|---|---|---|
| emilkowal.ski | design engineer, light, ultra-minimal | whole story in ~2 viewports, one type size, hierarchy purely from weight/color |
| paco.me | design engineer, **dark** | the direct counter-example: same near-black canvas, but a 3-column grid composes the space — density without noise |
| leerob.com | engineer/writer, dark serif | one-viewport homepage, prose + underlined links, zero boxed UI |
| rauno.me | interaction designer | confident composition; craft signal without widgetry |
| karpathy.ai | researcher | substance-first archetype; plain page, all content |

Screenshots: `reports/screenshots/wave6-audit/` (live site + references).

## What wave 5 got right (verified, kept)

- The modular type *scale* is real and consistent — no stray clamp()/font-black remained.
- Dark graphite + indigo restraint, Fraunces/Space Grotesk identity: distinctive, worth keeping.
- Content quality is **better than every reference** — honest metrics, engineering
  story-lines, provenance discipline. The material is not the problem.
- A11y rigor (axe 0, correct outline) and the zero-eager-cost heat toy architecture.

## The actual gap

Wave 5 fixed **scale** (how big things are). What it never addressed is **composition**
(how the page occupies space and time) and **conviction** (how many visual devices it
needs to feel credible). Specifically:

### 1. The page is long and empty where every reference is short and dense

6,090px tall at 1440 (≈6.8 viewports); ~8,600px on mobile (~10 screens, ~40% of it
Experience). Every reference tells its whole story in 1–2.5 viewports. On a near-black
canvas the cost is doubled: unfilled viewport reads as *void*, not luxury. The
whitespace-to-content ratio is inverted relative to the reference set — they are dense
inside short pages; this site is sparse inside a long one.

### 2. One narrow column floats in the dark — nothing composes the canvas

Every section is the same centered single column; at 1440 the right half of the screen is
permanently empty. paco.me solves the identical dark canvas with a multi-column grid;
leerob.com keeps the page to one viewport so the void never accumulates. Here, monotone
column + 6 vertical repeats = the "improvised, not gridded" feel GG keeps reacting to.

### 3. Alignment discipline breaks at the first scroll (bug)

Hero is `max-w-3xl`, every other section `max-w-4xl` — the hero's left edge sits ~64px
inboard of every section below it. The very first scroll breaks the grid.

### 4. Seven typographic voices where references use two

Fraunces display, long Fraunces *italic* passages (about lead + full research abstract —
italic is a display voice, unreadable at paragraph length), Space Grotesk body, JetBrains
Mono (metrics, chips, counters, datelines), tracked-caps eyebrows (0.3em, everywhere),
decorative section numerals, and a green status pill (a **second accent** next to indigo).
Every reference: one or two voices, hierarchy from weight and color. The site talks in
costumes.

### 5. Boxed-UI everywhere; references use none

Outlined cards, a 10-chip mono tag wall, a pill badge, four boxed hero buttons, a boxed
heat-toy trigger, a bordered `pip install` chip. On dark, 3.38:1 borders render as gray
noise — "widget," not "editorial." None of the five references draws a single box.

### 6. The carousel is the wrong pattern for the audience (and has real defects)

Nine products behind five clicks — a recruiter scans, never drives. Defects verified
in-browser: a **visible native scrollbar** across the track (premium killer); flagship and
secondary slides share one fixed track so secondary cards carry ~300px of dead interior;
cards clip mid-card at the viewport edge; Warmer alone has no dateline (private repo) so
card headers are asymmetric. Every reference lists work flat and scannable in one pass.
Wave 5's reviewer praised the carousel's *implementation* craft; the *pattern* is still
wrong for this audience.

### 7. Redundancy and self-reference in the copy

Hero tagline ≈ About lead (nearly the same sentence twice within 800px). Heat-toy intro
copy renders **twice** when activated (annex copy + the component's own copy — bug,
`products.tsx:74` + `heat-toy.tsx:137`). Metric labels speak internal jargon as huge
tracked caps ("HINGLISH EMBEDDING EVAL (SPEARMAN CORRELATION)"). Provenance-speak leaks
to visitors ("no placeholder, no guessed date"). Confidence is plain language.

### 8. Interaction layer: decorative or broken at the edges

- Command palette: no backdrop dim, visible inner scrollbar; the floating "⌘K Search" chip
  renders on **mobile/touch** where it is useless and overlaps the hero stats at 390px. A
  5-anchor single page does not need a jump menu — this is craft cosplay, +3.1KB eager.
- Reveal-on-scroll leaves every section `opacity: 0` until JS + intersection fire: the page
  below the hero is **invisible** in no-JS/reader-mode/print/full-page capture (verified —
  full-page screenshots render black).
- Count-up stats and the monogram draw-in are entrance decoration; no reference animates
  anything on entry.

## Diagnosis, in one sentence

The information *order* is right and the content is excellent, but the page is a set of
bordered widgets floating one-per-screen down a dark void, speaking in seven voices —
the fix is a **composition rebuild** (short, dense, grid-composed, box-free, two-voice),
not another token pass.

## Rebuild directives (wave 6)

1. **Length**: ≤3 viewports desktop (~2,800px), ≤5,000px mobile. Density, not stretching.
2. **Grid**: one container (max-w-5xl), one shared left edge; desktop sections compose as
   label-column + content-column (paco-style) so the canvas is *occupied*; mobile stacks.
3. **Type**: two voices — Fraunces (name, section heads, product names) + Space Grotesk
   (everything else); mono strictly for numerals/data; italics never at paragraph length;
   eyebrow caps at most once per screen; kill decorative numerals and the green pill.
4. **No boxes**: hierarchy from spacing/weight/color; links are underlined text; chips,
   cards, pills, boxed buttons eliminated (heat-toy input keeps its functional field).
5. **Work**: flat, scannable — 3 flagships as generous editorial entries (metric + story
   inline), 6 more as a compact index (incl. tracegauge, out of the footer); no carousel.
6. **Cut**: command palette (+its chip), count-up, monogram draw-in JS, reveal-on-scroll
   (or replace with no-JS-safe CSS), Now-strip stays dead, shipping log stays dead.
7. **Copy**: merge hero tagline/About lead; plain-language metric labels; delete
   provenance-speak from visitor-facing text (provenance.md still backs every number).
8. **Keep**: live data (product count, puzzle #, tracegauge downloads, freshness), heat toy
   as the one signature interaction (annexed to Warmer), all content accuracy, dark+indigo
   identity, a11y bar, ≤215KB budget (expect to land *lower* than wave 5's 204.6KB).
